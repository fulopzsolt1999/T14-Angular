Lépések a projekt létrehozásában

Az alábbi lépések voltak a projekt fontosabb állomásai, az elkészítése során.

  1. Alap elemek elkészítése
    1. Projekt létrehozása a ng new KisebbAngularFunkciok --no-standalone paranccsal
    2. Komponensek létrehozása az ng g c KompononsNeve paranccsal
    3. létrehoztam a kezdolap, hiba404, data-binding, ngif, ngfor és loggolo komponenseket
    4. Elkészíteni a routing részt az app-routing.modules.ts állományban.
    5. Telepíteni a bootstrap-et a npm install bootstrap paranccsal
    6. angular.json állományban a style és a script elemek módosítása/bővítése
      • "./node_modules/bootstrap/dist/css/bootstrap.css"
      • "./node_modules/bootstrap/dist/js/bootstrap.js"
    7. app.component.html állomány teljes tartalmának törtlése kivéve az router-outlet tag-et a végén
    8. Navigációs sáv kódjának beillesztése:https://getbootstrap.com/docs/5.3/components/navbar/ oldalról
    9. Elhelyezni a routerLink elemeket a navigációs sáv linkjeihez!
  2. Data Data-Binding
    • Interpoláció bemutatása: {{'{{}}'}} -ek közé elhelyezzük az elem nevét ami a backend kódban van
      • backend kód: szoveg: string = "Szeretem a programozást"
      • frontend kód: {{'{{szoveg}}'}}
    • Property binding bemutatása: []-ek közé helyezzük az attributumot amit a propertyvel módosítani akarunk, és létrehozunk neki egy változó értéket, aminek szeretnénk hogy az értékét megkapja
      • backend kód:kepCimke: string = "Meglepetés"; vagy épp kepCel: string = "_blank"; stb...
      • frontend kód:<a [href]="kepUtvonal" [target]="kepCel"><img [src]="kepLink" [alt]="kepAlternativSzoveg" [title]="kepCimke" [width]="kepSzelesseg"></a>
    • Event binding : ()-ek közé helyezzük az event nevét, mondjuk click event és hozzárendelünk egy függvényt[metódust], az angular backend kódból.
      • backend kód: {{'uzenet(): void { alert("Hello ez itt egy team14 projekt!"); '}}
      • frontend kód: (click)="uzenet()
    • Kétirányú adatkötés azaz Two Way DataBinding
      • Ha kétirányú adatkötésnél formokat használunk, akkor importálni kell a FormsModule elemet a app.module.ts állomány imports részénél, lásd az aktuális projektben is megtettük ezt!
      • backend kód: egyszerű változó létrehozása aOldal: number = 1;
      • frontend kód: hivatkozni [(ngModel)] elemmel: <input type="number" class="form-control w-100 my-2" id="aOldal" [(ngModel)]="aOldal">
      • Így elérjük, hogy az űrlapelemben módosított érték a backend kódban és módosuljon, és ha a backend kódon módosítanánk, az megjelenjen a frontend részen is!
  3. ngIf elem használata
    • Alapvetően, ha elemhez rendeljük az *ngIf értéket, akkor true logikai érték esetén megjeleniti azt, false esetén pedig elrejti.
    • frontend kód: <p *ngIf="szovegMegjelenit">ngIf teszt</p>
    • backend kód: szovegMegjelenit: boolean = true;
  4. ngFor elem használata
    • *ngFor segítségével ciklus vezérlési szerkezettel, járhatunk be egy összetett adatszerkezetet, legyen az akár tömb elemi értékekkel feltöltve, vagy épp objektumokkal töltött tömb.
    • backend kód (létrehozzuk az adatszerkezetet): bevasarloLista: string[] = ["tej", "kenyér", "hercz szalámi", "sör", "alma", "maci sajt", "csokis tej"]
    • Listában megjelentítjük az adatokat, ngFor és interpoláció segítségével az alábbi módon
    • frontend kód: {{'<li *ngFor="let termek of bevasarloLista"> {{termek}} </li>'}}
    • A bevásárló lista elemein megy végéig, úgy hogy minden aktuális emelre, "termék" néven hivatkozik
  5. Loggoló funkció készítése:
    • Létrehozzuk a változót amiben tároljuk az elemeket majd: bevasarloLista: string[] = [];
    • Készítünk egy elemet, amiben az akutálisan letárolandó értéket tároljuk: ujTermek!: string;
    • Majd végül egy függvény ami az eddig elemekehez hozzáadja, az új elemet: {{'termekHozzaad(): void { this.bevasarloLista.push(this.ujTermek); }'}}
    • Elkészítünk egy formot, ami 2 way databing segítségével , megjelenítni az elemet, illetve ha a formban lévő gombra kattintunk, feltölti az elemet, az összetett adatszerkezetünkben, a függvény meghívásával (event binding)
    • Végül, a feltöltött adatokat, *ngFor segítségével, dinamikusan megjelenítjük az oldalunkon
  6. Hibakezelő oldal
    • Beillesztem az alábbi szöveget az oldalra:<h1 class="text-center">Hiba:404, a keresett oldal nem található</h1>
